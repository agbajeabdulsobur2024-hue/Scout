# Scout — AI Market Intelligence Companion, built on 0G

Scout is a crypto market companion that explains what's happening in the
market and why — not a signal bot, not a price predictor. Ask it things
like *"why is BTC moving right now"* or *"what are the best setups right
now"* and it answers using fresh market data and a model running on **0G
Compute's decentralized inference network**.

Built for **The Zero Cup 2026**, starting June 18, 2026.

## Why 0G is load-bearing here, not a bolt-on

Every explanation Scout gives is generated by a real-time call through 0G
Compute (`app/zg_compute.py`) — there is no other LLM in this codebase, no
cached/templated answers, and no fallback to a normal hosted API. If 0G
Compute is down, Scout says so and stops, rather than quietly substituting
something else (see `main.py`'s startup health check). Remove 0G and Scout
has no reasoning layer left — it's the thing that actually does the work,
not a wrapper around something that already worked without it.

## How it works

```
User asks a question (Telegram)
        ↓
market_data.py   — fetches live price/volume/funding from public exchange APIs
        ↓
reasoning.py      — builds context from that data, computes a transparent
                     0-100 signal-strength score (plain arithmetic, not an
                     LLM guess — see below), and asks 0G Compute to explain it
        ↓
zg_compute.py    — the only place this app talks to a model, via 0G's
                     OpenAI-compatible proxy endpoint
        ↓
Answer sent back to the user
```

### A note on "confidence" / scoring

You'll see a signal-strength score (0-100) attached to opportunities. That
number is **plain, inspectable arithmetic** — volume spike vs. recent
average, position within the recent price range, magnitude of 24h move —
computed in `market_data.signal_strength()`, not invented by the model.
0G Compute explains what the score means; it never makes the score up.
We did this deliberately: a model hallucinating a confident-sounding
number is worse than no number at all.

## What's real right now vs. planned

**Built and working:**
- Live market data layer (Binance public REST — price, volume, 15m
  structure, funding rate)
- Deterministic signal-strength scoring
- Reasoning layer that explains a symbol or ranks current opportunities,
  entirely through 0G Compute
- Telegram chat interface with light intent routing (no slash commands
  required for natural questions)

**Planned next (stretch goal, not yet built):**
- 0G Storage as the actual memory layer — conversations, past
  explanations, and trade-relevant context persisted to and retrieved
  from 0G Storage, so "what did you tell me about ETH last week" is a
  real lookup against decentralized storage, not a local cache.

We're not claiming Storage integration exists until it does — see
`ARCHITECTURE.md` for the honest current-state diagram.

## Setup

1. Set up and start the 0G Compute sidecar — follow `SETUP_0G.md` (one-time wallet/funding step, then `cd zg-sidecar && npm install && npm start`, leave it running)
2. In a separate terminal: `pip install -r requirements.txt`
3. Create a Telegram bot via [@BotFather](https://t.me/BotFather), put the token in `.env`
4. `python main.py`
